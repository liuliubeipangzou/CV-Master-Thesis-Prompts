硕士毕业论文写作 Prompts

本项目整理了一系列针对计算机视觉（Computer Vision）方向硕士毕业论文写作的 **高阶 Prompt**。旨在利用大模型（LLM）作为你的**资深科研合作者**，辅助完成从选题、大纲、正文写作到深度润色的全过程。

## 💡 使用建议 (Best Practices)
1.  **Role-Play (角色设定)**：高质量的 Prompt 通常包含明确的角色设定（如“资深博导”），这能显著提升输出的专业度。
2.  **Constraints (约束条件)**：明确告知模型“不要做什么”（如“不要使用口语化表达”），比只说“要做什么”更有效。
3.  **Iterative Refinement (迭代优化)**：不要期待一次生成完美结果。通过追问（"请更学术一点"、"这个逻辑有点跳跃"）来打磨内容。
4.  **Academic Integrity (学术诚信)**：AI 是助手而非作者。核心观点、数据和创新点必须源于你的真实工作。务必人工核查所有生成的文献引用和事实描述。

## 1. 选题与大纲生成

**Prompt (选题灵感):**
```text
**Role:** 你是一位计算机视觉领域的资深博导，同时也是硕士学位论文盲审专家。你对 CVPR/ICCV/ECCV 的最新研究热点（如 Transformer, Diffusion Models, NeRF, MLLM 等）有深刻理解，并且非常清楚硕士毕业论文的“工作量”和“创新性”要求。

**Task:** 我是 CV 方向的硕士生，研究兴趣在 **[具体领域，如：弱监督语义分割]**。我希望解决 **[核心难点，如：标注成本高、边界模糊]**。请结合近 3 年的顶会趋势，为我推荐 3-5 个选题。

**Constraints:**
1. **可行性优先**：选题必须适合硕士生在 1 年内完成，避免需要 Google/Meta 级别算力（如预训练大模型）的题目。
2. **创新性适度**：不需要颠覆性的理论创新，可以是现有方法的改进、模块的组合创新或新场景的应用。
3. **工作量饱满**：每个选题必须包含足够的工作量（如：设计了新模块 + 做了大量对比实验），以满足毕业要求。

**Output Format:**
请为每个选题提供以下结构化信息：
- **题目 (Title)**：学术规范的暂定题目。
- **研究背景与意义 (Background & Significance)**：为什么做这个？
- **核心创新点 (Methodology)**：拟提出的具体改进方案（Method）。
- **预期实验结果 (Expected Results)**：在哪个数据集上验证？预期达到什么效果？
- **难点预警 (Difficulty Warning)**：可能遇到的最大坑是什么？
```

**Prompt (大纲生成):**
```text
**Role:** 资深学术写作顾问，擅长构建逻辑严密的论文框架。

**Task:** 我已确定论文题目为 **[题目]**。请为我生成一份详细的硕士毕业论文大纲。

**Requirements:**
1. **结构完整**：包含标准章节（绪论、相关工作、核心方法章节、实验结果与分析、总结与展望）。
2. **细化程度**：细化到三级标题（例如：3.2.1 多尺度特征提取模块）。
3. **内容指引**：在每个小节标题下，简要列出该小节应该写的**关键点 (Key Points)** 和 **逻辑流 (Logical Flow)**。
4. **工作量体现**：如果核心方法较多，建议拆分为两个章节（例如：第三章 改进算法A，第四章 改进算法B），以体现工作量的饱满。
```

## 2. 论文标题生成

**Prompt:**
```text
**Role:** 资深中文期刊编辑，对标题的学术性、简洁性和吸引力有极高要求。

**Task:** 我的论文提出了 **[核心方法]** 用于解决 **[核心任务]** 中的 **[核心问题]**。实验表明在 **[数据集]** 上提升了 **[指标]**。请生成 10 个符合中文学术规范的论文标题。

**Constraints:**
1. **拒绝标题党**：避免使用“震惊”、“完美”等非学术词汇。
2. **格式规范**：
   - 格式 A（标准型）：基于[方法]的[任务]研究
   - 格式 B（问题导向型）：面向[场景/问题]的[任务]方法研究
   - 格式 C（融合型）：融合[技术A]与[技术B]的[任务]算法
3. **关键词突出**：标题必须包含任务名称和核心技术关键词。

**Output Format:**
- 直接列出 10 个标题，按推荐程度排序。
```

## 3. 摘要写作 (Abstract)

**Prompt (中文摘要生成):**
```text
**Role:** 资深学术写作专家，擅长高度概括和提炼核心观点。

**Task:** 请帮我撰写计算机视觉硕士论文的中文摘要。

**Input Information:**
- **背景**：**[简述领域背景]**
- **问题**：**[现有方法在具体场景下的缺陷]**
- **方法**：**[本文提出的方法名称，以及 Step 1, Step 2...]**
- **结果**：**[在哪个数据集上，指标提升了多少]**

**Constraints:**
1. **逻辑严谨**：严格遵循“背景-问题-方法-结果-意义”的五段式逻辑。
2. **字数控制**：400-500 字左右（硕士论文摘要通常较长）。
3. **突出贡献**：明确指出本文的创新点（使用了“提出”、“设计”、“改进”等动词）。

**Output Format:**
- **摘要正文**：一段完整的中文摘要。
- **关键词**：3-5 个核心关键词。
```

**Prompt (英文摘要生成 - 非翻译):**
```text
**Role:** CVPR/ICCV 论文作者，英语母语者。

**Task:** 基于上述中文摘要的核心逻辑，为我撰写对应的英文摘要 (Abstract)。

**Constraints:**
1. **拒绝中式英语**：不要逐字翻译中文摘要！请重新组织语言，使其符合英语母语者的表达习惯。
2. **术语规范**：使用 CV 领域的标准术语（如 state-of-the-art, ablation study, end-to-end, robust）。
3. **时态规范**：
   - 描述方法、架构、结论：使用**一般现在时** (e.g., We propose, The method consists of...)。
   - 描述具体的实验过程：使用**一般过去时** (e.g., We evaluated, The model achieved...)。

**Output Format:**
- **Abstract**: The English abstract text.
- **Keywords**: 3-5 keywords.
```

## 4. 引言写作 (Introduction)

**Prompt (逻辑扩写):**
```text
**Role:** 严谨的逻辑学家和学术写作教练。你擅长使用“漏斗式”结构（Funnel Structure）将宏大的背景逐步收敛到具体的创新点。

**Task:** 请基于我提供的核心逻辑点，扩充并润色成一篇约 1-1.5 页的引言（Introduction）。

**Input Key Points:**
1. **大背景**：**[领域的重要性]**
2. **挑战与 Gap**：**[现有方法 A 的局限]**；**[现有方法 B 的局限]**。
3. **本文动机**：受 **[某种机制/现象]** 启发，我们思考...
4. **本文贡献**：(1) 提出了 **[核心算法]**；(2) 设计了 **[核心模块]**；(3) 在 **[数据集]** 上验证了有效性。

**Constraints:**
1. **逻辑衔接**：段落之间必须有流畅的过渡（Transition），避免碎片化。
2. **批判性思维**：在分析相关工作时，要客观指出其不足，为引出本文方法做铺垫，但不要恶意贬低。
3. **引用规范**：使用 [?] 占位符来标记需要插入引用的位置。

**Output Format:**
- 按段落输出引言内容。
- 在每个段落前，简要标注该段的**功能**（例如：【第一段：背景引入】）。
```

## 5. 国内外研究现状 (Related Work)

**Prompt (综述与批判):**
```text
**Role:** 领域综述专家，对该领域的历史沿革和最新进展了如指掌。

**Task:** 请撰写关于 **[子领域]** 的 Related Work 章节。需要涵盖 **[方法类别1]** 和 **[方法类别2]**。

**Constraints:**
1. **拒绝罗列**：不要写成“A做了这个，B做了那个”的流水账。必须进行**分类**和**比较**。
2. **Gap Analysis (核心)**：对于提到的每个经典/SOTA方法，简述其贡献后，必须**客观指出其在解决[你的核心问题]时的局限性**。这是为了证明你工作的必要性。
3. **时效性**：侧重于近 3-5 年的方法，但也要提及经典的奠基性工作。

**Output Format:**
- **分类 1 (Title)**: [内容]
- **分类 2 (Title)**: [内容]
- **总结 (Summary)**: 总结现有工作的共性问题，并用一句话引出本文方法的优势。
```

**Task:** 我设计了一个 **[模块名称]**，逻辑如下：**[自然语言描述计算过程，例如：先做卷积，再做Softmax，最后乘回原特征]**。请将其转化为规范的数学表达和 LaTeX 公式。

**Constraints:**
1. **符号定义**：必须在使用公式前清晰定义每个变量（如 $X \in \mathbb{R}^{C \times H \times W}$）。

3. **推导严谨**：公式之间的推导过程要连贯，不要跳步。

**Output Format:**
- **文字描述**：包含符号定义的学术文本。
- **LaTeX 公式**：可直接复制的代码块。

## 7. 实验结果与分析 (Results & Analysis)

**Prompt (深入分析):**
```text
**Role:** 资深数据分析师 & 研究员，擅长从枯燥的数据中挖掘深层原因。

**Task:** 基于以下实验数据，写一段深入的实验结果分析（Results Analysis）。

**Input Data:**
[在此处描述你的实验数据对比，例如：我的方法 mAP 提升了 2.5%，但在小目标上提升了 5.0%]

**Analysis Direction:**
1. **拒绝流水账**：不要只报数字（No number reporting only）。
2. **归因分析 (Why)**：解释为什么会有提升？建议方向：[例如：引入了多尺度特征融合 / 增强了边缘信息提取 / 抑制了背景噪声]。
3. **Case Study**：分析在 **[具体场景，如：遮挡/暗光/运动模糊]** 下表现更好的原因，证明模型的鲁棒性。

**Output Format:**
- **总体表现 (Overall Performance)**：概括主要指标提升。
- **归因分析 (Attribution Analysis)**：深度解析提升原因。
- **场景分析 (Scenario Analysis)**：针对特定场景的讨论。
```

## 8. 图表与数据描述 (Figures & Data Description)

**Prompt (图表描述 Caption):**
```text
**Role:** 细致的观察者，擅长撰写自包含（Self-contained）的图注。

**Task:** 我有一张 **[图表类型，如：网络架构图 / 实验对比柱状图 / 可视化热力图]**。图表显示了：**[描述图表内容]**。请写一段学术规范的图注（Caption）和正文引用描述。

**Constraints:**
1. **自包含性**：Caption 必须足够详细，读者不看正文也能大概理解图表含义。
2. **引导性**：正文描述要引导读者关注图表的关键信息（如趋势、峰值、对比差异），而不是机械复述数据。

**Output Format:**
- **Figure Caption**: [Figure 1. The architecture of...]
- **In-text Reference**: [As shown in Figure 1, ...]
```

## 9. 结论与展望 (Conclusion)

**Prompt:**
```text
**Role:** 资深学术编辑。

**Task:** 重写我的结论部分，使其更加精炼且有深度。

**Input Content:**
[粘贴你的结论草稿]

**Constraints:**
1. **避免重复**：不要直接复制摘要或引言的句子，要进行改写（Rephrase）。
2. **升华价值**：不仅要总结做了什么，还要强调工作的意义。
3. **具体展望**：未来方向要具体，拒绝空泛（如“我们将继续优化性能”是空泛的，应改为“我们将探索将其迁移到 3D 任务中”）。

**Output Format:**
- **总结 (Summary)**：概括核心贡献。
- **局限性 (Limitations)**：诚实但委婉地指出不足。
- **未来工作 (Future Work)**：2-3 个具体的改进方向。
```

## 10. 内容扩写与深度润色 (Content Expansion & Deep Polishing)

**Prompt (硕士论文内容扩写 - 丰满化):**
```text
**Role:** 你是一位注重细节的硕士生导师。你深知硕士论文需要通过详实的描述来体现工作量和理论深度，拒绝“干瘪”的论述。

**Task:** 请根据我提供的【核心要点】或【简短草稿】，扩写成一段**内容饱满、逻辑严密、符合硕士学位论文规范**的学术段落。

**Input:** [核心要点/关键词/简短句子]

**Constraints:**
1.  **细节填充（体现工作量）**：不要只重复观点，要主动补充技术细节、理论依据或实验参数。例如，提到“模型训练”时，补充优化器、学习率策略等细节。
2.  **逻辑显化**：在要点之间显式地补充因果推导（Because... Therefore...），使审稿人能顺畅跟随你的思路。
3.  **学术升维**：将口语化描述转化为规范的学术表达（如将“把图片变清楚”扩写为“通过引入残差模块有效恢复了图像的高频纹理信息”）。
4.  **篇幅扩展**：在不注水的前提下，将内容扩充至原来的 2-3 倍，使其在版面上看起来更加厚实。

**Output Format:**
- **扩写后文本 (Expanded Text)**：一段可以直接放入硕士论文正文的成熟段落。
- **扩写策略 (Expansion Strategy)**：简要说明在哪里补充了细节，以及如何体现工作量。
```

**Prompt (硕士论文深度润色 - 专家版):**
```text
**Role:** 你是一位经验丰富的硕士生导师，同时也是负责学位论文盲审的专家。你深知硕士毕业论文与期刊论文的区别：硕士论文更强调工作量的饱满度、逻辑的完整性以及写作的规范性。

**Task:** 请对提供的【中文硕士论文段落】进行专业审视与润色。你的目标是帮助学生顺利通过盲审和查重。

**Constraints:**
1. **修正阈值（学位论文导向）**：
   - **必须修改**：口语化表达（如“我们觉得”）、明显的语病、逻辑断层、标点符号错误。
   - **谨慎修改**：如果原文逻辑通顺且没有明显错误，**不要**为了追求“高大上”而随意删减篇幅。硕士论文需要足够的字数来支撑工作量，过度的精简反而可能被判定为工作量不足。
2. **查重友好（降重策略）**：
   - 在润色时，尽量使用多样化的句式和词汇。如果原文使用了过于通用的模板句（容易导致高查重率），请在保持原意的前提下进行改写（Rephrase）。
3. **语体规范（学位论文风）**：
   - **语气谦逊严谨**：使用“本文提出”、“实验结果表明”等客观陈述。避免使用过于绝对或夸张的形容词（如“完美解决”、“彻底消除”）。
   - **术语统一**：确保专业术语在全文中的一致性。
4. **格式适配（Word 友好）**：
   - 纯净文本：输出结果必须是纯文本，方便直接粘贴到 Word 中。严禁使用 Markdown 加粗、斜体。
   - 标点规范：严格使用中文全角标点符号。
5. **输出格式（分情况处理）**：
   - **Part 1 [润色后文本]**：
     * 直接输出润色后的内容。
   - **Part 2 [修改说明与建议]**：
     * 简要说明修改了什么（例如：“修正了口语化表达”、“调整了语序以降低查重风险”）。
     * 如果原文有逻辑漏洞或工作量描述不足的问题，请作为导师给出具体的补充建议。
   - 除以上两部分外，不要输出任何多余的对话。

**Execution Protocol:**
在输出前，请自我反问：
1. 修改后的文字是否符合硕士学位论文的规范？
2. 我是否误删了体现学生工作量的细节？
3. 我的改写是否有助于降低潜在的查重率？

**Input:** [在此处粘贴你的中文论文段落]
```

**Prompt (去口语化):**
```text
**Role:** 严谨的中文学术编辑。

**Task:** 下面这段话读起来比较口语化，不够专业。请用计算机视觉领域的专业学术词汇和句式对其进行润色，使其更加正式、流畅，增强逻辑性。

**Output Format:**
1. **润色后内容 (Polished Content)**：直接提供修改后的学术文本。
2. **修改说明 (Modification Notes)**：简要说明修改了哪些非专业表达或句式（例如：将“用了”改为“采用”，将“效果变好”改为“性能显著提升”）。

**Input:** [粘贴你的中文段落]
```

**Prompt (逻辑连贯性检查):**
```text
**Role:** 逻辑学家。

**Task:** 检查以下段落的逻辑流（Logical Flow），并对其进行重写以修复逻辑漏洞。

**Output Format:**
1. **重写后内容 (Rewritten Content)**：逻辑通顺、连接紧密的文本。
2. **逻辑诊断 (Logic Diagnosis)**：指出原段落存在的逻辑跳跃、指代不明或因果不当之处，并解释如何修复。

**Input:** [中文段落]
```

## 11. Reviewer 视角预审 (Pre-Review)

**Prompt (模拟盲审专家):**
```text
**Role:** 苛刻的硕士学位论文盲审专家（Reviewer）。你对“工作量是否饱满”、“创新点是否成立”、“实验对比是否公平”有极高的敏感度。

**Task:** 请阅读我的论文摘要和引言（或核心方法章节），并给出评审意见。

**Evaluation Criteria:**
1. **创新性 (Novelty)**：创新点是否足够硕士毕业要求？是否只是简单的模块堆砌（A+B）？
2. **完整性 (Completeness)**：实验对比是否充分？Baseline 选择是否合理？消融实验是否能证明模块有效性？
3. **写作质量 (Writing)**：逻辑是否严密？是否存在低级错误？

**Output Format:**
- **优点 (Strengths)**：2-3 点肯定。
- **缺点 (Weaknesses)**：3-5 点**尖锐**的批评（请不要客气，直接指出可能导致盲审不过的问题）。
- **改进建议 (Actionable Suggestions)**：针对每个缺点给出具体的修改方案。
```
