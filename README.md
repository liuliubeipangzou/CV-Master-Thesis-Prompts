硕士毕业论文写作 Prompts

本项目整理了一系列针对计算机视觉（Computer Vision）方向硕士毕业论文写作的 **高阶 Prompt**。旨在利用大模型（LLM）作为你的**资深科研合作者**，辅助完成从选题、大纲、正文写作到深度润色的全过程。

## 💡 使用建议 (Best Practices)
1.  **Role-Play (角色设定)**：高质量的 Prompt 通常包含明确的角色设定（如“资深博导”），这能显著提升输出的专业度。
2.  **Constraints (约束条件)**：明确告知模型“不要做什么”（如“不要使用口语化表达”），比只说“要做什么”更有效。
3.  **Iterative Refinement (迭代优化)**：不要期待一次生成完美结果。通过追问（"请更学术一点"、"这个逻辑有点跳跃"）来打磨内容。
4.  **Academic Integrity (学术诚信)**：AI 是助手而非作者。核心观点、数据和创新点必须源于你的真实工作。务必人工核查所有生成的文献引用和事实描述。

## 1. 选题与大纲生成

**Prompt (选题灵感):**
```text
**Role:** 你是一位计算机视觉领域的资深博导，同时也是硕士学位论文盲审专家。你对 CVPR/ICCV/ECCV 的最新研究热点（如 Transformer, Diffusion Models, NeRF, MLLM 等）有深刻理解，并且非常清楚硕士毕业论文的“工作量”和“创新性”要求。

**Task:** 我是 CV 方向的硕士生，研究兴趣在 **[具体领域，如：弱监督语义分割]**。我希望解决 **[核心难点，如：标注成本高、边界模糊]**。请结合近 3 年的顶会趋势，为我推荐 3-5 个选题。

**Constraints:**
1. **可行性优先**：选题必须适合硕士生在 1 年内完成，避免需要 Google/Meta 级别算力（如预训练大模型）的题目。
2. **创新性适度**：不需要颠覆性的理论创新，可以是现有方法的改进、模块的组合创新或新场景的应用。
3. **工作量饱满**：每个选题必须包含足够的工作量（如：设计了新模块 + 做了大量对比实验），以满足毕业要求。

**Output Format:**
请为每个选题提供以下结构化信息：
- **题目 (Title)**：学术规范的暂定题目。
- **研究背景与意义 (Background & Significance)**：为什么做这个？
- **核心创新点 (Methodology)**：拟提出的具体改进方案（Method）。
- **预期实验结果 (Expected Results)**：在哪个数据集上验证？预期达到什么效果？
- **难点预警 (Difficulty Warning)**：可能遇到的最大坑是什么？
```

**Prompt (大纲生成):**
```text
**Role:** 资深学术写作顾问，擅长构建逻辑严密的论文框架。

**Task:** 我已确定论文题目为 **[题目]**。请为我生成一份详细的硕士毕业论文大纲。

**Requirements:**
1. **结构完整**：包含标准章节（绪论、相关工作、核心方法章节、实验结果与分析、总结与展望）。
2. **细化程度**：细化到三级标题（例如：3.2.1 多尺度特征提取模块）。
3. **内容指引**：在每个小节标题下，简要列出该小节应该写的**关键点 (Key Points)** 和 **逻辑流 (Logical Flow)**。
4. **工作量体现**：如果核心方法较多，建议拆分为两个章节（例如：第三章 改进算法A，第四章 改进算法B），以体现工作量的饱满。
```

## 2. 论文标题生成

**Prompt (硕士学位论文标题):**
```text
**Role:** 资深高校学位论文评审专家，熟悉《研究生学位论文撰写规范》。

**Task:** 我的论文提出了 **[核心方法]** 用于解决 **[核心任务]** 中的 **[核心问题]**。实验表明在 **[数据集]** 上提升了 **[指标]**。请生成 10 个符合中文硕士学位论文规范的标题。

**Constraints:**
1.  **规范性**：标题应准确、简明、具体，一般不超过 25 个汉字。
2.  **避免口语**：严禁使用“一种...”、“基于...”的简单堆砌，尝试更具学术深度的表达（如“面向...的...研究”、“...关键技术研究”）。
3.  **体现工作量**：标题应暗示这是一个系统性的工作，而非单一的小技巧（例如使用“系统”、“框架”、“方法研究”等词汇）。
4.  **格式参考**：
    -   **面向场景型**：面向 [应用场景] 的 [核心任务] 方法研究
    -   **技术融合型**：基于 [技术A] 与 [技术B] 的 [核心任务] 算法研究
    -   **问题驱动型**：[核心问题] 下的 [核心任务] 关键技术研究

**Output Format:**
- 直接列出 10 个标题，按推荐程度排序。
```

## 3. 摘要写作 (Abstract)

**Prompt (中文摘要生成 - 硕士版):**
```text
**Role:** 资深学术写作专家，擅长撰写逻辑严密、内容饱满的学位论文摘要。

**Task:** 请帮我撰写计算机视觉硕士学位论文的中文摘要。

**Input Information:**
- **背景**：**[简述领域背景]**
- **问题**：**[现有方法在具体场景下的缺陷]**
- **方法**：**[本文提出的方法名称，以及 Step 1, Step 2...]**
- **结果**：**[在哪个数据集上，指标提升了多少]**

**Constraints:**
1.  **五段式逻辑**：严格遵循“背景与意义 - 问题与挑战 - 核心方法（分点阐述） - 实验结果 - 总结与展望”的结构。
2.  **篇幅控制**：**800-1000 字**（注意：硕士论文摘要通常比小论文摘要更长，需要详细阐述方法细节和实验结论）。
3.  **分点阐述**：在介绍方法时，使用“首先，...；其次，...；最后，...”的句式，清晰展示 2-3 个创新点（Innovation Points），体现饱满的工作量。
4.  **客观严谨**：使用“本文提出”、“实验验证”等客观主语，避免过度夸大。

**Output Format:**
- **摘要正文**：一篇结构完整、细节丰富的中文摘要。
- **关键词**：3-5 个核心关键词，用分号隔开。
```

**Prompt (英文摘要生成 - 硕士版):**
```text
**Role:** 英语母语的学术编辑，熟悉 CV 领域术语。

**Task:** 基于上述中文摘要，撰写对应的英文摘要 (Abstract)。

**Constraints:**
1.  **去中式英语**：不要逐字翻译！请使用符合英语母语习惯的学术表达（如使用长短句结合、定语从句等）。
2.  **时态规范**：
    -   Background & Method: Present Tense (e.g., This thesis proposes...)
    -   Experiments & Results: Past Tense (e.g., The experiments showed that...)
3.  **术语准确**：使用标准的 CV 术语 (e.g., state-of-the-art, robust, generalization ability)。
4.  **长度适配**：英文摘要长度应与中文摘要相匹配，通常为 1-1.5 页。

**Output Format:**
- **Abstract**: The English abstract text.
- **Keywords**: 3-5 keywords, separated by semicolons.
```

## 4. 引言写作 (Introduction)

**Prompt (逻辑扩写 - 硕士版):**
```text
**Role:** 严谨的逻辑学家和学术写作教练。你擅长使用“漏斗式”结构（Funnel Structure）将宏大的背景逐步收敛到具体的创新点。

**Task:** 请基于我提供的核心逻辑点，扩充并润色成一篇约 3-5 页的硕士论文第一章（绪论）。

**Input Key Points:**
1.  **大背景**：**[领域的重要性]**
2.  **挑战与 Gap**：**[现有方法 A 的局限]**；**[现有方法 B 的局限]**。
3.  **本文动机**：受 **[某种机制/现象]** 启发，我们思考...
4.  **本文贡献**：(1) 提出了 **[核心算法]**；(2) 设计了 **[核心模块]**；(3) 在 **[数据集]** 上验证了有效性。
5.  **章节安排**：简述后续章节的内容。

**Constraints:**
1.  **逻辑衔接**：段落之间必须有流畅的过渡（Transition），避免碎片化。
2.  **批判性思维**：在分析相关工作时，要客观指出其不足，为引出本文方法做铺垫，但不要恶意贬低。
3.  **引用规范**：使用 [?] 占位符来标记需要插入引用的位置。
4.  **章节导引 (Thesis Organization)**：在引言的最后，必须包含一段“本文组织结构”，简要介绍各章（第2章到第X章）的主要内容。

**Output Format:**
- 按段落输出引言内容。
- 在每个段落前，简要标注该段的**功能**（例如：【第一段：背景引入】、【最后一段：论文组织结构】）。
```

## 5. 国内外研究现状 (Related Work)

**Prompt (综述与批判 - 硕士版):**
```text
**Role:** 领域综述专家，对该领域的历史沿革和最新进展了如指掌。

**Task:** 请撰写关于 **[子领域]** 的“国内外研究现状”章节（通常为第二章）。需要涵盖 **[方法类别1]** 和 **[方法类别2]**。

**Constraints:**
1.  **结构化综述**：不要写成流水账。请按照 **[分类标准]** 将现有方法分为几类（例如：基于传统方法的... vs 基于深度学习的...），并分小节阐述。
2.  **国内外兼顾**：如果可能，请区分“国内研究现状”和“国外研究现状”，或者在综述中提及国内外的代表性工作（这在中文硕士论文中很常见）。
3.  **Gap Analysis (核心)**：对于提到的每个经典/SOTA方法，简述其贡献后，必须**客观指出其在解决[你的核心问题]时的局限性**。这是为了证明你工作的必要性。
4.  **时效性**：侧重于近 3-5 年的方法，但也要提及经典的奠基性工作。

**Output Format:**
- **2.1 [子领域] 国内外研究现状**
    - **2.1.1 [分类 1] 研究现状**：[内容]
    - **2.1.2 [分类 2] 研究现状**：[内容]
- **2.2 本章小结**：总结现有工作的共性问题，并用一句话引出本文方法的优势。
```

## 6. 方法论描述 (Methodology)

**Prompt (数学公式化 - 硕士版):**
```text
**Role:** 算法工程师兼应用数学家，擅长将直观的算法思想转化为严谨的数学形式。

**Task:** 我设计了一个 **[模块名称]**，逻辑如下：**[自然语言描述计算过程，例如：先做卷积，再做Softmax，最后乘回原特征]**。请将其转化为规范的数学表达和 LaTeX 公式。

**Constraints:**
1.  **符号定义表**：请先给出一份核心符号定义表（Symbol Table），确保全文符号一致（如 $X$ 始终代表输入，$Y$ 代表输出）。
2.  **公式规范**：使用标准的 LaTeX 语法。
3.  **推导严谨**：公式之间的推导过程要连贯，不要跳步。对于复杂的推导，提供必要的文字解释。
4.  **伪代码**：如果算法流程复杂，请附带一份标准的 Algorithm 伪代码。

**Output Format:**
- **符号定义**：核心变量说明。
- **文字描述**：包含公式引用的学术文本。
- **LaTeX 公式**：可直接复制的代码块。
- **Algorithm 伪代码**：标准的伪代码块。
```

**Prompt (网络架构整体描述 - 硕士版):**
```text
**Role:** 系统架构师。

**Task:** 请用学术语言描述我的 **[网络名称]** 整体架构（通常是第三章或第四章的开头）。

**Input:**
- 输入：[尺寸]
- Backbone：[名称]
- Neck：[功能]
- Head：[功能]

**Constraints:**
1.  **总-分结构**：先用一段话概括整体流程（Overall Workflow），再分小节详细介绍各个子模块。
2.  **图文对照**：假设有一张“图 3-1 整体架构图”，请在文中多次引用该图（“如图 3-1 所示...”），描述数据流（Data Flow）如何在各模块间传递。
3.  **模块功能**：明确每个模块的输入、输出以及核心功能。

**Output Format:**
- **3.1 整体架构概述**：[文本]
- **3.2 [子模块1] 详细设计**：[文本]
- **3.3 [子模块2] 详细设计**：[文本]
```

## 7. 实验结果与分析 (Results & Analysis)

**Prompt (深入分析 - 硕士版):**
```text
**Role:** 资深数据分析师 & 研究员，擅长从枯燥的数据中挖掘深层原因。

**Task:** 基于以下实验数据，写一段深入的实验结果分析（Results Analysis）。

**Input Data:**
[在此处描述你的实验数据对比，例如：我的方法 mAP 提升了 2.5%，但在小目标上提升了 5.0%]

**Analysis Direction:**
1.  **SOTA 对比分析**：与近 3 年的 SOTA 方法进行对比，强调在哪些指标上取得了优势（尤其是核心指标）。
2.  **消融实验 (Ablation Study) 分析**：(非常重要) 分析去掉某个模块后性能下降的原因，从而证明该模块的有效性。这是硕士论文工作量的重要体现。
3.  **归因分析 (Why)**：解释为什么会有提升？建议方向：[例如：引入了多尺度特征融合 / 增强了边缘信息提取 / 抑制了背景噪声]。
4.  **定性分析 (Qualitative Analysis)**：描述可视化结果（如：“从图 4-3 可以看出，本文方法生成的边缘更清晰...”）。

**Output Format:**
- **4.3.1 对比实验分析**：[SOTA 对比文本]
- **4.3.2 消融实验分析**：[模块有效性分析文本]
- **4.3.3 可视化结果分析**：[定性分析文本]
```

## 10. 内容扩写与深度润色 (Content Expansion & Deep Polishing)

**Prompt (硕士论文内容扩写 - 丰满化):**
```text
**Role:** 你是一位注重细节的硕士生导师。你深知硕士论文需要通过详实的描述来体现工作量和理论深度，拒绝“干瘪”的论述。

**Task:** 请根据我提供的【核心要点】或【简短草稿】，扩写成一段**内容饱满、逻辑严密、符合硕士学位论文规范**的学术段落。

**Input:** [核心要点/关键词/简短句子]

**Constraints:**
1.  **细节填充（体现工作量）**：不要只重复观点，要主动补充技术细节、理论依据或实验参数。例如，提到“模型训练”时，补充优化器、学习率策略等细节。
2.  **逻辑显化**：在要点之间显式地补充因果推导（Because... Therefore...），使审稿人能顺畅跟随你的思路。
3.  **学术升维**：将口语化描述转化为规范的学术表达（如将“把图片变清楚”扩写为“通过引入残差模块有效恢复了图像的高频纹理信息”）。
4.  **篇幅扩展**：在不注水的前提下，将内容扩充至原来的 2-3 倍，使其在版面上看起来更加厚实。

**Output Format:**
- **扩写后文本 (Expanded Text)**：一段可以直接放入硕士论文正文的成熟段落。
- **扩写策略 (Expansion Strategy)**：简要说明在哪里补充了细节，以及如何体现工作量。
```

**Prompt (硕士论文深度润色 - 专家版):**
```text
**Role:** 你是一位经验丰富的硕士生导师，同时也是负责学位论文盲审的专家。你深知硕士毕业论文与期刊论文的区别：硕士论文更强调工作量的饱满度、逻辑的完整性以及写作的规范性。

**Task:** 请对提供的【中文硕士论文段落】进行专业审视与润色。你的目标是帮助学生顺利通过盲审和查重。

**Constraints:**
1. **修正阈值（学位论文导向）**：
   - **必须修改**：口语化表达（如“我们觉得”）、明显的语病、逻辑断层、标点符号错误。
   - **谨慎修改**：如果原文逻辑通顺且没有明显错误，**不要**为了追求“高大上”而随意删减篇幅。硕士论文需要足够的字数来支撑工作量，过度的精简反而可能被判定为工作量不足。
2. **查重友好（降重策略）**：
   - 在润色时，尽量使用多样化的句式和词汇。如果原文使用了过于通用的模板句（容易导致高查重率），请在保持原意的前提下进行改写（Rephrase）。
3. **语体规范（学位论文风）**：
   - **语气谦逊严谨**：使用“本文提出”、“实验结果表明”等客观陈述。避免使用过于绝对或夸张的形容词（如“完美解决”、“彻底消除”）。
   - **术语统一**：确保专业术语在全文中的一致性。
4. **格式适配（Word 友好）**：
   - 纯净文本：输出结果必须是纯文本，方便直接粘贴到 Word 中。严禁使用 Markdown 加粗、斜体。
   - 标点规范：严格使用中文全角标点符号。
5. **输出格式（分情况处理）**：
   - **Part 1 [润色后文本]**：
     * 直接输出润色后的内容。
   - **Part 2 [修改说明与建议]**：
     * 简要说明修改了什么（例如：“修正了口语化表达”、“调整了语序以降低查重风险”）。
     * 如果原文有逻辑漏洞或工作量描述不足的问题，请作为导师给出具体的补充建议。
   - 除以上两部分外，不要输出任何多余的对话。

**Execution Protocol:**
在输出前，请自我反问：
1. 修改后的文字是否符合硕士学位论文的规范？
2. 我是否误删了体现学生工作量的细节？
3. 我的改写是否有助于降低潜在的查重率？

**Input:** [在此处粘贴你的中文论文段落]
```

**Prompt (去口语化):**
```text
**Role:** 严谨的中文学术编辑。

**Task:** 下面这段话读起来比较口语化，不够专业。请用计算机视觉领域的专业学术词汇和句式对其进行润色，使其更加正式、流畅，增强逻辑性。

**Output Format:**
1. **润色后内容 (Polished Content)**：直接提供修改后的学术文本。
2. **修改说明 (Modification Notes)**：简要说明修改了哪些非专业表达或句式（例如：将“用了”改为“采用”，将“效果变好”改为“性能显著提升”）。

**Input:** [粘贴你的中文段落]
```

**Prompt (逻辑连贯性检查):**
```text
**Role:** 逻辑学家。

**Task:** 检查以下段落的逻辑流（Logical Flow），并对其进行重写以修复逻辑漏洞。

**Output Format:**
1. **重写后内容 (Rewritten Content)**：逻辑通顺、连接紧密的文本。
2. **逻辑诊断 (Logic Diagnosis)**：指出原段落存在的逻辑跳跃、指代不明或因果不当之处，并解释如何修复。

**Input:** [中文段落]
```

**Prompt (去 AI 味/去机械感 - 硕士论文版):**
```text
**Role:** 资深学术编辑，专注于纠正“AI 腔”和“翻译腔”，熟悉中文硕士学位论文的语言规范。

**Task:** 下面这段话读起来有明显的 AI 生成痕迹（逻辑连接词堆砌、词汇空洞、句式重复）或翻译腔。请将其重写为**地道、简练、逻辑紧密**的中文学术表达，符合硕士毕业论文的写作风格。

**Diagnosis & Constraints:**
1.  **剔除“正确的废话”**：删除“显而易见”、“众所周知”、“随着...的快速发展”等无实质信息量的垫话。
2.  **拒绝连接词滥用**：严禁使用“首先...其次...再次...最后...”的僵硬排比。尽量用逻辑本身的递进（如因果、转折、并列）来衔接句子，而不是靠生硬的连接词。
3.  **去“翻译腔”**：
    *   **拆分长句**：避免英语式的长定语堆砌（如“一个基于...的具有...特征的...方法” -> 拆分为短句）。
    *   **主动语态**：少用被动句（“被提出”），多用无主句或主动句（“本文提出”、“实验采用”）。
4.  **实词替换虚词**：
    *   禁用：“起到了关键作用”、“具有重要意义”、“全面的分析”等万金油词汇。
    *   改用：“显著提升了”、“解决了...问题”、“深入分析了...原因”等具体动词。
5.  **审慎修改**：如果原文已经足够自然、专业，且没有明显的 AI 味，**请保持原样**，不要为了修改而强行改写，以免破坏原意或引入新的问题。

**Output Format:**
1.  **重写后文本 (Humanized Version)**：一段自然流畅、符合硕士论文语感的文字。
2.  **主要修改点 (Key Changes)**：简要列出 2-3 个具体的修改细节（如：删除了“此外”，将长句拆分，替换了空洞形容词）。

**Input:** [AI 生成的段落]
```

## 11. Reviewer 视角预审 (Pre-Review)

**Prompt (模拟盲审专家 - 硕士版):**
```text
**Role:** 苛刻的硕士学位论文盲审专家（Reviewer）。你对“工作量是否饱满”、“创新点是否成立”、“格式是否规范”有极高的敏感度。

**Task:** 请阅读我的论文摘要、引言或核心方法章节，并给出模拟评审意见。

**Evaluation Criteria (学位论文标准):**
1.  **工作量 (Workload)**：本文的工作量是否达到硕士毕业要求？是否存在“拼凑”嫌疑？
2.  **创新性 (Novelty)**：创新点是否明确？是简单的 A+B，还是有深度的改进？（硕士论文不要求颠覆性创新，但要求有明确的改进和验证）。
3.  **规范性 (Standardization)**：语言是否学术？图表是否清晰？逻辑是否闭环？

**Output Format:**
- **评审结论**：[通过 / 修改后通过 / 不通过]（模拟）
- **主要优点 (Strengths)**：[2-3 点]
- **主要问题 (Weaknesses)**：[3-5 点尖锐批评，如：工作量单薄、实验对比不充分、逻辑不通]
- **修改建议 (Actionable Suggestions)**：[针对每个问题的具体整改方案]
```

## 12. 答辩准备 (Defense)

**Prompt (答辩 PPT 大纲 - 硕士版):**
```text
**Role:** 金牌演讲指导教练，擅长设计引人入胜的学术汇报逻辑。

**Task:** 基于我的论文内容，生成一份 15-20 分钟的硕士毕业答辩 PPT 大纲。

**Constraints:**
1.  **时间分配**：重点在“方法”和“实验”部分（占 60% 时间），背景和现状快速带过。
2.  **逻辑线索**：每一页 PPT 都要围绕“发现问题 -> 解决问题 -> 验证效果”的逻辑展开。

**Output Format:**
请按页生成大纲，每页包含：
- **页码与标题**：(e.g., Page 3: 研究背景与意义)
- **核心要点 (Bullet Points)**：该讲什么（控制在 3-4 点）。
- **演讲提示 (Speaker Notes)**：这页的重点话术。
```

```text
**Role:** 毕业答辩委员会主席，以提问刁钻著称。

**Output Format:**
- **Q1**: [问题]
    - **回答策略**：[先肯定问题价值，再从...角度回答]
- **Q2**: ...
```
